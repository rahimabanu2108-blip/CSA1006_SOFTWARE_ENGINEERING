# CSA1006_SOFTWARE_ENGINEERING
# Telemedicine Scrum Project

## Overview

This repository contains a Scrum-based project management demonstration for a **Telemedicine Mobile Application**. The project was completed as part of an Agile Project Management assessment to understand how Scrum is used to manage software development and handle changing requirements.

A Trello board was created to simulate the Scrum workflow, including the Product Backlog, Sprint Backlog, task progress, and sprint completion.

---

## Problem Statement

During Sprint Planning, the Product Owner frequently changes the Product Backlog. These continuous changes create confusion among the development team, reduce productivity, and make it difficult to achieve sprint goals.

This project analyzes the issue and proposes suitable Scrum practices to improve backlog management and team collaboration.

---

## Objectives

* Understand Scrum roles and responsibilities.
* Analyze the causes of requirement instability.
* Improve Product Backlog prioritization.
* Apply Scrum concepts using a Trello board.
* Recommend improvements through Sprint Retrospectives.

---

## Scrum Workflow

The Scrum workflow followed in this project is:

Stakeholders → Product Owner → Product Backlog → Sprint Planning → Sprint Backlog → Development → Sprint Review → Sprint Retrospective

---

## Trello Board

The project was managed using Trello with the following workflow:

* Product Backlog
* Sprint Backlog
* In Progress
* Testing
* Done

Tasks were moved through these stages to represent the progress of a sprint.
